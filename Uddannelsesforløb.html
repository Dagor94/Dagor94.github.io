Uddannelsesforløb

Datatekniker med speciale i programmering

På data- og kommunikationsuddannelsen skifter man mellem at have undervisning på skolen og at være i virksomhedspraktik. Specialet i programmering tager afsæt i at udvikle avancerede systemer og forskellige typer applikationer, som er både drifts- og sikkerhedstestet. Specialet omfatter både programmering af hardware, komplekse databaseløsninger og avancerede produkter til særlige formål som f.eks. lagerstyring, handel eller statistiske beregninger.

Uddannelsen varer typisk 4½ år, ekskl. grundforløbet, hvoraf lærlingen er på skole i 55 uger.

    Hovedforløb 1

    August 2023 - Oktober 2023 - (10 uger)

    Karakter: 11.0 gennemsnit


    Fokuset på 1. hovedforløb er at gøre eleverne mere komfortable med at kode, etablere grundlæggende koncepter inden for programmering og lærer om databaser. Dette gives til udtryk i tværfagligheden på hovedforløbet da flere fag går på tværs af hinanden.


    Fag:


  • Grundlæggende Programmering 

    • Faget fokuserede på at give en bred forståelse af programmeringskoncepter og -værktøjer, så man er i stand til at udvikle effektive og pålidelige programmer, såsom;

    • - Git & versionsstyring

    • - Debugging

    • - Unit testing

    • - Clean code

    • - Generel koding

  • Objektorienteret Programmering (1/2) 

    • Praktisk erfaring og dybere forståelse af OOP-koncepter til udvikling af komplekse programmer og softwareløsninger. Emnerne som blev gennemgået var:

    • - Klasser

    • - Objekter

    • - Exeception Handling

    • - Indkapsling

    • - Polymorfi

    • - Nedarvning

    • - Acess Modifiers

    • - Override og Overload metoder

  • Clientsideprogrammering (1/2) 

    • Grundlæggende forståelse af web-programmering og -udvikling med færdigheder til at bygge en simpel webside. Emner gennemgået var:

    • - HTML Syntaks

    • - CSS

    • - Box Modellen

    • - Bootstrap

    • - Client Server Akitektur

  • Databasprogrammering (1/3) 

    • Grundlæggende forståelse for at programmere med relationelle databaser og færdigheder i oprettelse, administration og forespørgsler med T-SQL. Fik kendskab til følgende elementer;

    • - T-SQL

    • - Tabeller, kolonner & felter

    • - CRUD-implementering i C#

    • - Nøgler og relationer

    • - Aggregering, subqueries og Joins

  • Serverteknologi 1 - Databaseserver 

    • Færdigheder til at administrere og optimere en Microsoft SQL-server og dens databaser, der var fokus på driften af en server. Fik kendskab til følgende elementer;

    • - Introduktion til Relationelle Databaser

    • - Administration af brugeres rettigheder og adgang

    • - SQLCMD - SQL Syntaks introduktion

    • - Backups og Recovery - Fokus på backup strategier og metoder

    • - Metadata, Monitorering og Optimering af en database

    • - Views og Stored Procedures

  • Netværk 

    • Faget var delt op i to dele - Infrastruktur og Programmering.
    Infrastrukturdelen havde fokus på:

    • - Repetition af OSI-modellen, IPV4 og DHCP

    • - Opbygning af et segmenteret neværk

    • - VLAN og NAT

    Programmeringsdelen fokuserede på kodning af applikationer til "Netværksfaget" - hovedsageligt hvordan man fik et program til at kunne Subnette IPv4 addresser. Der var fokus på følgende:

    • - WPF - C# teknologi

    • - Subnetting

    • - Generel Kodestruktur og benyttelse af OOP

    • - Talsystemer og talkonvertering

  • Computerteknologi 

    • Fokus på Docker som teknologi og virtualisering generelt, med henblik på hvordan det virker, hvordan man bruger det og hvorfor det er blevet en industri standard i mange virksomheder i dag.

    • - Virtualisering og Containere

    • - Docker

    • - Docker Compose

    • - Docker Images

    • - Docker Network

    • - Introduktion til RAID, Docker Swarm og Kubernetes

  • Databasprogrammering (2/3)

  • GUI - Programmering (1/2) 

    • Fokus vil være på programmering af en grafisk brugerflade - enten WPF eller Avalonia - med baggrund i en MVVM opbygning.

  • Clientsideprogrammering (2/2) 

    • Dette bygger videre på faget fra H1, dog med fokus på JavaScript fremfor HTML og CSS. Formålet er at kunne benytte JavaScript til at give en webside funktionalitet, og benytte JQuery og AJAX til at kommunikere med en backend web service.
    faget har introduceret følgende emner:

    • - JQuery

    • - AJAX - Asynchronous JavaScript And XML

  • Serveradministration og sikkerhed 

    • Fokus på serveradministration i forhold til et client-serversystem. Dette fag er derfor fokus på AD domain services som DHCP og DNS samt hvordan man optimere sikkerheden på AD og applikationer på serveren. derudover vil følgende også blive lagt vægt på:

    • - Konfiguration af domæner og domæneskove og redegørelse for trusts og sites.

    • - Oprettelse og administration af bruger- og gruppekonti, samt tilladelser til forskellige ressourcer.

    • - Konfiguration af Backup, Recovery og replikering af Active Directory.

    • - Anvendelse af lokal sikkerhed på et netværk, herunder omdøbe konti, ændre konto-egenskaber, administration af brugerkonti herunder resette password og slette konti.

    • - Administration af en Member Server, lokale sikkerheds- og gruppepolitikker, herunder implementering, auditering og auditpolitik, sikkerhedslog, krypteret filesystem og anvendelse af hjælpeværktøjer til sikkerhedskonfiguration.

    • - Administration og Konfiguration af Domain Controllerens lokale sikkerhed, gruppe, domain og Site politikker og rangfølgen af implementerede politikker

  • Serverteknologi 2 

    • Faget har fokus på sikkerhed på serverniveau samt administration.
    Hovedsageligt følgende emner:

    • - Installere og konfigurere serversystem med netværkstjenester som DHCP, DNS, Firewall/Routing.

    • - Delegere DNS-zoner mellem flere servere.

    • - Opsætte fjernadgang og VPN på serveroperativsystemet.

    • - Konfigurere netværkssikkerhed med PKI, IPSec, Radius og lignende.

    • - Administrere storage og printning, samt brugertilladelser.

    • - Bruge værktøjer til administration, backup, overvågning og fejlretning.

    • - Forstå centrale cloudbaserede serverdriftsbegreber.

    • Her er der fokus på supportdelen af IT faget
    Faget arbejder med blandt andet:

    • - Redegørelse af de almindelige roller, der findes i og omkring 1st level support og 2nd level support.

    • - Service Requests opgaver - Hvad er det, hvordan benyttes det og hvordan optimere man det.

    • - Indsamling, vurdering, sortering og registrering af relevante symptomer og informationer ved fejl og henvendelser om hjælp.

    • - Problembeskrivelser ud fra analyse af objekter, indsamlede symptomer og gennemførte problemløsningsaktiviteter.

    • - Finde, vurdere og anvende eksisterende viden og dokumentation til problemløsning og support.

    Januar 2025 - Marts 2025 (10 uger)

    Karakter: XX


    Det helt store emne på 4. hovedforløb er Internet of Things (IoT) og embedded programmering. Det gennemsyre flere af fagene på hovedforløbet blandt andet IoT og Embeddede systemer og Linux. ligeledes med Big Data bliver koblet sammen.


    Fag:


  • Linux rettet mod server og Embedded 

    • Formålet med dette fag er et give indsigt i Linux systemer og miljø.

    Det vil blandt andet gennemgå følgende:

    • - Linux kommandoer.

    • - Linux System, shell- og programmeringsmiljø.

    • - forskellige værktøjer og programmeringsmodellen til Linux.

    • - Programmeringsmodellen til forskellige Linux varianter, herunder Linux Server og embedded Linux.

  • IoT og Embeddede Systemer 

    • Dette er det store fag på hovedforløbet - omkring 40% af hovedforløbet er dette fag. Formålet er at kunne redegøre for de forskellige begreber, teknikker, standarder, modeller og processer, der typisk indgår i IoT og IIoT løsninger.

    Emner som blandt andet vil blive berørt:

    • - Løsningsmuligheder ud fra konkrete cases på IoT/IIoT løsninger.

    • - Risici og udfordringer, der skal tages i betragtning ved IoT/IIoT.

    • - System- og integrationsmuligheder med IoT/IIoT på nuværende systemer.

    • - Kendte anvendelsesområder med IoT/IIoT.

    • - Opbygge en mindre prototype-løsning af et embedded system.

    • - Værktøjer til at kommunikere med et embedded system, og til håndtering af programudvikling.

    • - Styre input og output af data i et embedded system.

  • IoT og Embeddede Linux Systemer 

    • Fokus på Embedde Linux systemer

    Der vil være fokus på følgende:

    • - Installere og opsætte et Linux-baseret styresystem på en embedded platform.

    • - Grundlæggende filstruktur i det Linux-baserede styresystem

    • - Eksternt klientprogram som fx winscp og puTTY, til at arbejde på et Linux-baseret styresystem.

    • - Via Shell kommandoer, installere, opgradere og afinstallere software på et Linux-baseret styresystem

    • - Opsætte rettigheder og brugerprofiler på et Linux-baseret styresystem

    • - Udvikle applikation på et Linux-baseret embedded system.

    • - Opbygge og udvikle et IoT/IIoT-baseret system, som kan indlæse data fra sensorer som fx vision-sensorer, eller data fra intelligente sensorer, behandle dataene og videresende dem til et centralt system eller en skyløsning.

    • - Viden om sikkerhed, følge Best-Practice sikkerhedsanvisninger, og tage stilling til nødvendigheden af procedurer for løbende opdatering.
